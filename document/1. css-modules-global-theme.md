# CSS Modules và global theme class

## Vấn đề

Khi viết selector theo theme trong file CSS Module, các class global như `.dark-theme` hoặc `.light-theme` có thể bị CSS Modules scope thành class hash riêng của module.

Ví dụ trong `src/css/button.module.css`:

```css
.dark-theme .invert[data-hover][data-geist-button] {
  background: var(--themed-hover-bg, #383838);
}
```

Nhìn thì đúng, nhưng khi build có thể bị compile thành:

```css
.button-module__...__dark-theme .button-module__...__invert[data-hover][data-geist-button] {
  background: var(--themed-hover-bg, #383838);
}
```

Selector này không match DOM thật, vì theme class đang nằm trên `html`:

```html
<html class="dark-theme">
```

Kết quả là button không đổi `background` hoặc `color` theo theme, dù CSS trong source nhìn có vẻ đúng.

## Nguyên nhân

CSS Modules local-scope class selector trong file `.module.css`. Các class của component như `.button`, `.invert`, `.secondary` nên được scope. Nhưng các class global do theme provider gắn vào DOM, như `.dark-theme`, phải được đánh dấu là global.

Ngoài ra cần chú ý cascade. Hai selector giống nhau cùng set một property thì rule nằm sau sẽ override rule nằm trước:

```css
.invert[data-hover][data-geist-button] {
  background: var(--themed-hover-bg, #ccc);
}

.invert[data-hover][data-geist-button] {
  background: var(--themed-hover-bg, #383838);
}
```

Trong trường hợp này, fallback `#383838` sẽ luôn thắng, kể cả ở light theme.

## Cách xử lý

Dùng `:global(...)` cho class theme global:

```css
.invert[data-hover][data-geist-button],
.invert[data-active][data-geist-button] {
  background: var(--themed-hover-bg, #ccc);
}

:global(.dark-theme) .invert[data-hover][data-geist-button],
:global(.dark-theme) .invert[data-active][data-geist-button] {
  background: var(--themed-hover-bg, #383838);
}
```

Nếu selector có kết hợp với `:is(...)`, vẫn phải đánh dấu class theme là global:

```css
:is(
  :global(.dark-theme) .secondary.invert[data-hover][data-geist-button],
  :global(.dark-theme) .secondary.invert[data-active][data-geist-button]
) {
  background-color: var(--ds-gray-200);
}
```

## Checklist khi gặp lỗi theme trong CSS Module

1. Kiểm tra file có phải `.module.css` không.
2. Nếu selector dùng `.dark-theme`, `.light-theme`, `.dark`, hoặc class global từ provider, hãy bọc bằng `:global(...)`.
3. Kiểm tra cascade: có rule nào cùng specificity và nằm sau đang override không.
4. Build lại và inspect CSS output trong `.next/static/.../*.css`.
5. Selector đúng phải giữ class theme global, ví dụ:

```css
.dark-theme .button-module__...__invert { ... }
```

Không đúng nếu class theme cũng bị hash:

```css
.button-module__...__dark-theme .button-module__...__invert { ... }
```

## Ghi chú cho repo này

Theme provider đang gắn class theme lên `document.documentElement` (`html`) với mapping:

```js
{
  light: 'light-theme',
  dark: 'dark-theme',
}
```

Vì vậy trong CSS Modules, các selector phụ thuộc theme nên viết theo mẫu:

```css
:global(.dark-theme) .localClass { ... }
:global(.light-theme) .localClass { ... }
```
